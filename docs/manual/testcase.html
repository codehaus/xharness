TestCase

Description

TestCase is a container task that may contain any other Ant Task. It forms the logical unit of a "test". It works similar to the TestGroup Task ith the difference that it will stop executing it's containing Tasks and fail as soon as one contained Task fails. It also has an "owner" attribute that can be used to identify an owner (person) responsible for the TestCase.

Parameters

Attribute Description Required
name The name of the TestCase. Used for Result logging. No (*)
owner The owner of this TestCase. Defaults to "unknown". No

(*) If no name is specified, the default name is "testcase" for the first Task and "testcase_n" (n >= 2) for subsequent Tasks.

Example

  
      <testcase name="foo_test" owner="eng-orbix" skip="win">
          <echo message="hello World"/>
          <xh-javabg classname="test.Server"/>
          <xh-exec executable="client"/>
      </testcase>
