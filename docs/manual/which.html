Which

Description

Which is used in conjunction with an executable or file name and a PATH variable and returns the full path of the executable file name, similar to the UNIX "which" command.

Which utilises the ANT PathTokenizer function and so is suitable for either UNIX or Windows paths or a mixture of both. This means that we can use the separator characters "\" or "/" and the path separator characters ":" or ";"

Parameters

Attribute Description Required
executable The file to check for. Yes
path Path structure that contains the directories we will search. Yes
property The name of the property to set. Yes
failonerror Controls whether an error (such as not locating the file) stops the build or is merely reported to the screen. The default value (false) will not stop the build. No

On Windows platforms the Which Task automatically detects the windows typical file extensions for executables: ".com", ".exe" and ".bat" if they are not specified in the executable attribute.

Examples

  
      <which property="my.prop" executable="itlocator.exe"
             path="C:\Temp;C:\Program Files\IONA\asp\6.1\bin;Z:\tools">
Will set the property my.prop to e.g. C:\Program Files\IONA\asp\6.1\bin\itlocator.exe 
  
      <which property="my.prop" executable="itlocator"
             path="C:\Temp;C:\Program Files\IONA\asp\6.1\bin;Z:\tools">
Will also set the property my.prop to e.g. C:\Program Files\IONA\asp\6.1\bin\itlocator.exe 
  
      <which property="my.prop" executable="itlocator"
             path="/usr/bin:/usr/sbin:/usr/atria/bin:/local/products/IONA/asp/6.1/bin">
Will set the property my.prop to e.g. /local/products/IONA/asp/6.1/bin/itlocator
