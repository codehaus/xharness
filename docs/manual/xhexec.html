Xh-Exec

Description

Synchronously executes a system command.

Extends the Ant Exec Task. All it's parameters and nested elements (except the ones listed below) are supported.

Additional and modified Parameters

Attribute Description Required
failonerror Modified default value. Mark this process as failed, if the command exits with a returncode other than 0. Default is "true" No
timeout Modified default value. Enabled by default with a value of 1 hour. No
dir Modified default value. If the xh-exec Task is used inside an XHarness testsuite and the dir attribute is not explicitly specified, the dir is set to the value of the property current.test.dir. No
executable If the xh-exec Task is used inside an XHarness testsuite and the executable attribute does not denote an abolute path to the executable, the executable name is prepended with the value of the property current.test.dir. No

Additional parameters specified as nested elements

envset

Use a nested <EnvSet> element to pass a set of environment variables to the process.

Example:

    <envset id="config.env">
        <env key="IT_DOMAIN_NAME" value="filedomain"/>
        <env key="IT_PRODUCT_DIR" file="${product.dir}"/>
        <env key="IT_CONFIG_DOMAINS_DIR" file="${product.dir}/etc/domains"/>
        <env key="IT_LICENSE_FILE" file="${product.dir}/etc/licenses.txt"/>
    </envset>
  
    <!-- ... -->
    
    <xh-exec executable="itlocator">
       <envset refid="config.env"/>
    </xh-exec>
