XHarnessReport

Description

Merge the individual XML files generated by the XHarness task and apply a stylesheet on the resulting merged document to provide a browsable report of the test results similar to Ant's JUnitReport.

Parameters

Attribute Description Required
tofile The name of the XML file that will aggregate all individual XML test suite previously generated by the XHarness task. No. Default to TESTS-TestSuites.xml
todir The directory where should be written the file resulting from the individual XML test suite aggregation. No. Default to current directory
failonerror Stop the build, if failed tests are detected. Default is "false". No

Nested Elements

fileset

xharnessreport collects individual xml files generated by the XHarness task using the nested <FileSet> element.

report

Generate a browsable report based on the document created by the merge.

Parameters

Attribute Description Required
format The format of the generated report. Must be "noframes" or "frames". No, default to "frames"
styledir The directory where the stylesheets are defined. They must be conforming to the following conventions:
  • frames format: the stylesheet must be named frames.xsl.
  • noframes format: the stylesheet must be named noframes.xsl.
 No. Default to embedded stylesheets.
todir The directory where the files resulting from the transformation should be written to. No. Default to current directory

Example

      <xharnessreport todir="./reports">
          <fileset dir="./reports">
              <include name="*.xml"/>
          </fileset>
          <report format="frames" todir="./report/html"/>
      </xharnessreport>

would generate a TESTS-TestSuites.xml file in the directory reports and generate the default framed report in the directory report/html.
